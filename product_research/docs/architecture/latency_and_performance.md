# Требования к времени отклика и производительности

## Цель
Обеспечить **реалистичный и комфортный диалог** без ощутимых задержек. Длительные паузы разрушают эффект «живого клиента» и снижают вовлечённость.

## Ключевые метрики (SLA)

| Компонент | Метрика | Требование (MVP) | Целевое значение |
|----------|--------|------------------|------------------|
| ASR (Whisper) | Время распознавания 5 сек речи | ≤ 2 сек | ≤ 1 сек |
| LLM (Qwen) | Генерация ответа клиента (50 токенов) | ≤ 3 сек | ≤ 1.5 сек |
| TTS | Синтез 5 сек аудио | ≤ 1 сек | ≤ 0.5 сек |
| **Полный цикл** (менеджер говорит → ИИ отвечает) | End-to-end latency | **≤ 6 сек** | **≤ 3 сек** |

> ⚠️ Если задержка превышает 8 сек — система показывает уведомление: «Связь восстанавливается…» и не прерывает сессию.

## Обоснование
- Человек начинает ощущать «неловкую паузу» уже после **2 секунд** молчания в телефонном разговоре.  
- В реальном звонке клиент может сказать: «Алло? Вы меня слышите?» — если собеседник молчит дольше 6 сек.  
- Для массового использования (100+ одновременных сессий) система должна масштабироваться без деградации latency.

## Мониторинг
- Все задержки логируются по каждой сессии.  
- Алерт при среднем latency > 5 сек в течение 5 минут.  
- Еженедельный отчёт: «95-й перцентиль времени ответа».
